My Process

How I think with data

I always start with the question, not the query. Here's what that looks like in practice.

Step 01

Start with the decision

Before I open SQL Server or write a single line of code, I want to know: what decision is this analysis going to support? If I can't answer that clearly, I'm not ready to touch the data yet.

Step 02

Ask the harder question

It's easy to query what the data makes convenient. I try to go after the question that actually matters, even if the SQL gets ugly or the joins get complicated. Precision over convenience.

Step 03

Find the surprise

A good insight is not a summary of what everyone already knows. It's the thing that makes someone pause. The best findings I've delivered are the ones that challenged what stakeholders expected to see.

Step 04

Write for the decision-maker

The person reading my work probably didn't run the query. So I write for them: the manager, the VP, the executive. Clear recommendations, not just numbers. Analysis that ends with "here's what to do next."
