The Problem

Build a model that estimates sale price, and be honest about where it fails

Real estate agents needed a predictive pricing model estimating sale price from measurable property features, with clear guidance on trustworthiness. The dataset was 1,460 residential transactions from Ames, Iowa (Kaggle). 81 original features, price range $34,900–$755,000, median $163,000. Cleaned to 77 features before modeling.

The Insight

R² of 0.79 overall. R² of -0.08 for affordable homes. The model is worse than guessing for half the market.

Key Insight
"The model's overall accuracy was hiding a critical failure. R² of -0.08 for affordable homes is more useful than R² of 0.79 overall. It tells stakeholders exactly where not to trust the model."

An overall R² of 0.79 would normally be considered a success. But an A/B split by price segment told a completely different story. High-price homes: R² = 0.66, decent. Affordable homes (below $163K): R² = -0.08, the model was literally worse than guessing.

Affordable pricing is driven by location data (school district, crime rates, walkability) absent from the dataset. The aggregate metric was hiding a critical failure in half the market.

The Solution

Deploy selectively. Flag failures. Plan the next version.

Recommended deploying the model ONLY for properties above $163K median. Flag affordable listings for mandatory manual review. Version 2 must add school district, crime, and walkability data to close the affordable segment gap.

The most valuable output was not the model itself; it was the deployment boundary. Knowing where the model fails reshaped the entire recommendation from "deploy broadly" to "deploy only above $163K."

What I Learned

  • Aggregate metrics lie. R² of 0.79 made the model look production-ready. Segment-level analysis showed it would actively mislead agents for half their listings.
  • Knowing where a model fails is more valuable than knowing where it works. The -0.08 R² finding reshaped the entire deployment recommendation from "deploy broadly" to "deploy only above $163K."
  • Missing data is itself a finding. The affordable segment failure pointed directly to the features we needed next: location-based data the original dataset lacked.
